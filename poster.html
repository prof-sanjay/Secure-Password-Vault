Objective

  • Implement cryptographic hardening utilizing Argon2id for memory-hard key derivation to securely protect the primary access vault.
  • Integrate AES-GCM authenticated encryption to ensure data confidentiality and integrity checks for all stored credentials.
  • Develop a fault-tolerant recovery mechanism leveraging Shamir's Secret Sharing to mathematically distribute the master decryption key.
  • Distribute cryptographic shares across isolated Virtual Machines (VMs), systematically eliminating single points of failure within the backup infrastructure.
  • Evaluate algorithmic resilience against offline dictionary and brute-force attacks to validate the operational integrity of the security model.

Problem Statement

  • Single Point of Failure: Centralized vaults are permanently lost if the master key is forgotten.
  • Brute-Force Risk: Without rate-limiting, local vaults can be brute-forced.
  • Memory Leaks: Decrypted AES keys remaining in RAM pose a major session hijacking risk.

Implementation

  • Cryptographic Core: AES-256 for vault storage (`vault.json`) and Argon2id for resilient key derivation.
  • Secret Splitting: The Shamir’s Secret Sharing algorithm divides the master key into n distinct shares.
  • VM Broker Network: The application distributes the generated shares strictly across independent Virtual Machines, never locally.
  • Recovery Protocol: Automatically reconstructs the master key by querying VMs over secure TLS. Requires a threshold k to succeed.
  • Active Mitigations: Artificial login delays, and mathematically enforced 5-minute RAM wiping timeouts.

Results / Pros & Cons

  • Fault Tolerance: System survives hardware failures (up to n - k offline VMs) without losing data.
  • Zero-Knowledge: Neither the central server nor individual VMs can reconstruct the key autonomously.
  • Defense in Depth: Defends against network sniffing (HTTPS) and application layer attacks (auto-timeout).
  • Overhead Cost: Maintaining multiple VMs increases infrastructure and administrative complexity.
  • Network Dependency: Recovery is impossible without stable network access to reach the VM threshold.

Conclusion

This project successfully balances absolute local cryptographic security with practical usability. By integrating Shamir's Secret Sharing with a distributed Virtual Machine infrastructure, the system eliminates the devastating localized consequences of a forgotten master password. The multi-layered approach—combining Argon2id, AES-256, explicit RAM wiping, and network TLS—ensures the credential vault remains impenetrable to single-point compromises, delivering a robust solution for organizational and individual password management.
